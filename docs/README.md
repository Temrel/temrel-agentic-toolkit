# Docs

Per-item documentation that is too long to live in an item's own `README.md`: deep dives, design notes, walkthroughs, and reference material.

Each item keeps a short `README.md` next to its code. When an item needs more (a tutorial, a design rationale, screenshots), it goes here under a folder named after the item, and the item's README links to it.

## What belongs here

Long-form docs only. The one-line "what" and "why", the Temrel issue link, and the subscribe link always stay in the item's own README.
