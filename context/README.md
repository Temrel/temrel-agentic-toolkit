# Context engineering

One of the six elements of agentic engineering. This is the craft of shaping **what an agent knows**: the instructions, files, and structure (CLAUDE.md, system prompts, retrieved context) that an agent reads before it acts. Good context engineering means the agent has exactly what it needs and nothing that wastes tokens or misleads it.

## What belongs here

Tools that help you build, audit, trim, or generate agent context: context linters, prompt and instruction templates, retrieval helpers, token budgeters.

## Items

- [context-linter](./context-linter/) : paste a context file, get a bloat score and line-referenced trim suggestions, all in the browser.
